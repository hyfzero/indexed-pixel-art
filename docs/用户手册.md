# Indexed Pixel Art 用户手册

**语言：** [简体中文](用户手册.md) | [English](User-Guide.en.md)

本手册说明两种使用方式：

1. 直接运行 Python 工具，校验、渲染或修改已有像素画。
2. 把项目安装为 Codex Skill，用自然语言让 Codex 设计并生成像素画。

## 1. 功能概览

`indexed-pixel-art` 生成的不是经过滤镜处理的“像素风图片”，而是严格的索引色像素画：

- 每一个像素都对应一个调色板索引。
- 原图尺寸必须与声明的画布完全一致。
- 调色板冻结后不能添加临时颜色。
- 不使用图片生成模型、抗锯齿、自动量化或缩小普通图片。
- 渲染后会重新打开 PNG，检查尺寸、索引、颜色和透明度。
- 局部修改使用坐标 Patch，不必整张重画。
- 二值图可以导出为嵌入式设备使用的 C 数组。

## 2. 环境准备

需要：

- Python 3.11 或更高版本
- Pillow
- jsonschema
- pytest
- PyYAML

在项目目录安装依赖：

```powershell
python -m pip install -r requirements.txt
```

如果 Windows 中 `python` 命令不可用，请安装 Python，并在安装时启用
“Add Python to PATH”。

验证工具是否可用：

```powershell
python scripts/validate_pixel_art.py --help
python scripts/render_pixel_art.py --help
```

## 3. 快速体验

项目提供三个示例：

| 示例 | 画布 | 调色板 |
|---|---:|---:|
| `binary-heart-16x16.json` | 16×16 | 黑白二色 |
| `binary-cat-16x16.json` | 16×16 | 黑白二色 |
| `four-color-slime-16x16.json` | 16×16 | 四色 |

校验心形示例：

```powershell
python scripts/validate_pixel_art.py examples/binary-heart-16x16.json
```

成功时会显示 `VALID`、画布尺寸、调色板数量和像素总数。

渲染原图和 8 倍预览：

```powershell
python scripts/render_pixel_art.py examples/binary-heart-16x16.json `
  --output output/binary-heart.png `
  --preview-scale 8
```

macOS/Linux shell 使用反斜杠续行：

```bash
python scripts/render_pixel_art.py examples/binary-heart-16x16.json \
  --output output/binary-heart.png \
  --preview-scale 8
```

输出文件：

```text
output/binary-heart.png
output/binary-heart_preview_8x.png
```

原图仍然是 16×16。预览图只是使用最近邻算法放大到 128×128。

## 4. 使用命令行工具

### 4.1 校验 JSON

```powershell
python scripts/validate_pixel_art.py <源文件.json>
```

增加非阻断风格检查：

```powershell
python scripts/validate_pixel_art.py <源文件.json> --style-check
```

硬性错误会返回非零退出码，例如：

- 行数或行宽错误
- 调色板索引不连续
- 像素引用未声明颜色
- 调色板超过 `palette_limit`
- 透明索引未声明

风格问题默认只产生警告，例如孤立像素、主体触碰边界、明显不对称或视觉重心失衡。

### 4.2 渲染 PNG

```powershell
python scripts/render_pixel_art.py <源文件.json> `
  --output <原图.png> `
  --preview-scale 8
```

可选参数：

- `--style-check`：渲染后打印风格警告。
- `--silhouette-preview`：额外输出剪影预览。
- `--preview-scale N`：设置预览放大倍数，范围为 1 到 64。

渲染前会执行完整校验。渲染后还会逐像素检查 PNG；检查失败时不会保留错误原图。

### 4.3 检查 PNG

建议始终同时提供源 JSON：

```powershell
python scripts/inspect_png.py output/binary-heart.png `
  --source examples/binary-heart-16x16.json
```

该命令检查：

- PNG 是否为索引色 `P` 模式
- 实际尺寸是否正确
- 实际使用了哪些索引和颜色
- 是否存在额外颜色
- 是否存在意外透明度
- PNG 像素是否与源矩阵逐像素一致

### 4.4 局部修改

坐标以左上角为原点，从 `0` 开始：

```text
(0,0) ─────────→ x
  │
  │
  ↓
  y
```

创建 `patch.json`：

```json
{
  "base_revision": 1,
  "set": [
    { "x": 7, "y": 4, "color": 1 },
    { "x": 8, "y": 4, "color": 1 },
    { "x": 9, "y": 4, "color": 0 }
  ]
}
```

应用 Patch：

```powershell
python scripts/apply_patch.py examples/binary-heart-16x16.json patch.json `
  --output output/binary-heart-revision-2.json
```

工具会检查基础版本、坐标范围和颜色索引。任意操作非法时，整个 Patch 都会失败。

修改后重新渲染：

```powershell
python scripts/render_pixel_art.py output/binary-heart-revision-2.json `
  --output output/binary-heart-revision-2.png `
  --preview-scale 8
```

### 4.5 导出 C 数组

仅支持严格二值作品：

```powershell
python scripts/export_c_array.py examples/binary-heart-16x16.json `
  --output output/binary_heart.h
```

数据按逐行、从左到右、从上到下、MSB first 排列。宽度不足整字节时，每行在 C
数组中尾部补零；源 JSON 和原始 PNG 尺寸不会改变。

## 5. 创建自己的 JSON

可以复制现有示例作为起点。基本结构如下：

```json
{
  "version": "1.0",
  "name": "my-icon",
  "revision": 1,
  "mode": "indexed",
  "canvas": {
    "width": 16,
    "height": 16
  },
  "palette_limit": 2,
  "allow_transparency": false,
  "palette": [
    { "index": 0, "hex": "#FFFFFF", "name": "background" },
    { "index": 1, "hex": "#000000", "name": "foreground" }
  ],
  "encoding": "matrix",
  "pixels": [
    "0000000000000000"
  ],
  "style": {
    "preset": "binary_icon"
  }
}
```

注意：上面的 `pixels` 只是结构示意。16×16 作品必须提供恰好 16 行，并且每行恰好
16 个索引字符。

第一版仅支持索引 `0` 到 `9`。矩阵中不能写 HEX、RGB、空格或逗号。

## 6. 透明背景

默认禁止透明。需要透明背景时，必须显式声明：

```json
{
  "palette_limit": 2,
  "allow_transparency": true,
  "transparent_index": 0,
  "palette": [
    { "index": 0, "hex": "#FFFFFF", "name": "transparent" },
    { "index": 1, "hex": "#000000", "name": "foreground" }
  ]
}
```

此例仍是两个状态：透明和黑色。

如果同时需要透明、黑色和白色，则必须使用 `palette_limit: 3`。不支持半透明像素。

## 7. 安装为 Codex Skill

Codex 当前会从以下常用位置发现 Skill：

- 用户级：`$HOME/.agents/skills`
- 仓库级：`<目标仓库>/.agents/skills`

本项目根目录包含 `SKILL.md`，但放在任意普通目录时不会自动成为全局 Skill。需要把
整个项目目录复制或链接到上述位置之一。

### 7.1 Windows 用户级安装

假设本项目位于：

```text
D:\LLM_Proj\indexed-pixel-art
```

推荐创建目录联接，代码更新后 Skill 会同步更新：

```powershell
New-Item -ItemType Directory -Force "$HOME\.agents\skills" | Out-Null
New-Item -ItemType Junction `
  -Path "$HOME\.agents\skills\indexed-pixel-art" `
  -Target "D:\LLM_Proj\indexed-pixel-art"
```

不希望使用链接时，可以复制：

```powershell
Copy-Item -Recurse -Force `
  "D:\LLM_Proj\indexed-pixel-art" `
  "$HOME\.agents\skills\indexed-pixel-art"
```

### 7.2 macOS/Linux 用户级安装

```bash
mkdir -p "$HOME/.agents/skills"
ln -s "/absolute/path/to/indexed-pixel-art" \
  "$HOME/.agents/skills/indexed-pixel-art"
```

### 7.3 仓库级安装

如果只希望某个项目使用此 Skill，将整个目录放入：

```text
目标仓库/
└── .agents/
    └── skills/
        └── indexed-pixel-art/
            ├── SKILL.md
            ├── scripts/
            ├── schemas/
            └── ...
```

仓库级安装适合团队共享。提交 `.agents/skills/indexed-pixel-art` 后，其他成员在该仓库
中启动 Codex 时也能发现它。

安装或更新后，Codex 通常会自动检测变更。如果 Skill 没有出现在选择器中，请重启
Codex 或新建线程。

## 8. 在 Codex 中调用

Skill 可用于 Codex App、CLI 和 IDE 扩展。

### 8.1 确认 Skill 已发现

在 Codex CLI 或 IDE 中运行：

```text
/skills
```

也可以在输入框键入 `$`，检查列表中是否出现：

```text
$indexed-pixel-art
```

### 8.2 显式调用

推荐在提示词开头明确写出 Skill 名称：

```text
$indexed-pixel-art 画一个 16x16 黑白火箭图标。
不要透明背景，输出 JSON、原始 PNG 和 8 倍预览图。
```

Codex 会读取 `SKILL.md`，设计完整索引矩阵，调用校验器和渲染器，并复检 PNG。

### 8.3 隐式调用

也可以直接描述任务：

```text
画一个严格二值的 16x16 OLED 电池图标，并导出 C 数组。
```

任务与 Skill 的描述匹配时，Codex 可以自动选择该 Skill。为了保证一定使用，仍建议写
`$indexed-pixel-art`。

## 9. 推荐提示词

### 9.1 使用默认参数

```text
$indexed-pixel-art 画一个咖啡杯像素图标。
```

未指定参数时，Skill 默认使用：

- 16×16 原始画布
- 黑白二色
- 不透明背景
- 白色背景、黑色前景
- 8 倍最近邻预览

### 9.2 明确规格

```text
$indexed-pixel-art 创建一个 24x24 的四色宝箱 Sprite。
调色板固定为 #F4E8C1、#8C5A2B、#3B2414、#FFD34E。
不允许透明，使用 retro_sprite 风格。
输出源 JSON、原始 PNG、8 倍预览，并执行 style-check。
```

### 9.3 OLED 与 C 数组

```text
$indexed-pixel-art 创建一个 32x16 的严格二值 Wi-Fi 图标，
用于单色 OLED。禁止孤立像素，输出 PNG、预览图和 MSB-first C 数组。
```

### 9.4 透明 Sprite

```text
$indexed-pixel-art 创建一个 16x16 的黑色幽灵剪影。
使用两个状态：透明背景索引 0、黑色前景索引 1。
输出 JSON、原图和 8 倍预览。
```

### 9.5 局部修改

```text
$indexed-pixel-art 修改 output/cat.json：
把左耳顶部向上移动一个像素，其他位置保持不变。
请使用坐标 Patch，增加 revision，重新校验、渲染并检查 PNG。
```

### 9.6 风格整理

```text
$indexed-pixel-art 检查 examples/binary-cat-16x16.json 的轮廓。
只修复无意义孤立像素和不规则边缘台阶，不得改变画布和调色板。
使用 Patch 修改，并列出修改的坐标。
```

## 10. Codex 执行流程

调用 Skill 后，Codex 应按以下顺序工作：

1. 理解主题、用途和输出要求。
2. 确认画布尺寸、调色板数量和透明度。
3. 冻结调色板。
4. 先设计轮廓，再添加内部结构和负空间。
5. 写出完整索引矩阵 JSON。
6. 运行严格校验。
7. 修复全部硬错误后再次校验。
8. 渲染索引色原图和最近邻预览。
9. 重新打开 PNG，与源 JSON 逐像素核对。
10. 向用户返回源 JSON、原图、预览图以及检查结果。

如果用户要求局部修改，Codex 应优先生成 Patch，而不是重画整个矩阵。

## 11. 如何检查 Codex 是否正确使用了 Skill

完成后应至少看到：

- 一个源 JSON 文件
- 一个保持声明尺寸的原始 PNG
- 一个文件名包含 `_preview_8x` 的放大预览
- 校验输出包含 `VALID`
- 渲染输出包含 `RENDERED`
- PNG 检查输出包含 `Constraint status: PASS`

二值作品的实际索引应只有 `0, 1`。四色作品不得出现第五种索引。默认模式不应包含
透明度。

以下行为表示没有遵守 Skill：

- 只返回一张图片，没有源矩阵
- 原图尺寸被放大
- 边缘出现灰色抗锯齿
- 自动增加颜色
- 输入错误时静默补齐或裁剪
- 局部修改导致无关区域变化

## 12. 常见问题

### Codex 找不到 Skill

检查目录是否为：

```text
$HOME/.agents/skills/indexed-pixel-art/SKILL.md
```

或者：

```text
<仓库根目录>/.agents/skills/indexed-pixel-art/SKILL.md
```

确认文件夹名称和 `SKILL.md` 中的名称都是 `indexed-pixel-art`，然后重启 Codex。

### 显式调用后没有生成文件

提示词中明确要求 Codex 执行完整流程，并给出输出目录：

```text
$indexed-pixel-art 在当前项目的 output/ 中生成文件。
必须实际运行校验、渲染和 PNG 检查命令，不要只给方案。
```

### 风格警告是否代表失败

不是。颜色、尺寸、透明度和矩阵错误属于失败；孤立像素、轮廓节奏和视觉重心等风格
问题默认只是警告。

### 可以把现有照片转换成像素画吗

本 Skill 第一版不提供自动图片转换，也不会先生成普通图片再缩小。它从离散索引矩阵
开始设计每一个像素。

### 可以使用超过十种颜色吗

第一版不支持。矩阵使用单字符索引，因此仅支持 `0` 到 `9`。
