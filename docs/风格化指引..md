# 三、像素画风格化指引

系统不仅要保证像素矩阵合法，还需要生成具有明确风格、可读轮廓和像素画质感的作品。

风格化必须建立在离散像素矩阵之上。不得调用图片生成模型，不得先绘制普通图片再缩小，不得使用滤镜模拟像素效果。

## 3.1 风格规则的优先级

所有规则按照以下优先级执行：

```text
1. 画布尺寸约束
2. 调色板颜色约束
3. 透明度约束
4. 像素矩阵合法性
5. 主题可识别性
6. 风格化规则
7. 装饰性细节
```

当风格化规则与硬性约束冲突时，必须舍弃风格效果，而不是突破硬性约束。

例如：

```text
错误：
为了让边缘更平滑，增加灰色抗锯齿像素。

正确：
保持黑白二值，通过调整轮廓台阶改善观感。
```

---

## 3.2 风格配置字段

在 JSON 源文件中增加可选的 `style` 字段：

```json
{
  "style": {
    "preset": "clean_icon",
    "silhouette_priority": "high",
    "outline": "solid",
    "outline_index": 1,
    "shading": "none",
    "dithering": "disabled",
    "symmetry": "approximate_vertical",
    "detail_level": "low",
    "edge_language": "intentional_steps",
    "negative_space": "clear",
    "isolated_pixels": "avoid",
    "visual_weight": "balanced"
  }
}
```

第一版至少支持以下字段：

```text
preset
silhouette_priority
outline
outline_index
shading
dithering
symmetry
detail_level
edge_language
negative_space
isolated_pixels
visual_weight
```

字段可以由 LLM 读取并用于设计，但不得改变底层颜色数量和画布尺寸。

---

## 3.3 基础构图原则

生成像素画时，按照以下顺序设计：

```text
1. 确定主体占据的区域。
2. 确定重心和朝向。
3. 先画外轮廓剪影。
4. 检查缩小后是否仍然可识别。
5. 再添加内部结构。
6. 最后添加少量装饰像素。
7. 删除破坏可读性的冗余细节。
```

不要一开始就绘制零散细节。

对于较小画布，例如 `8x8`、`16x16` 或 `24x24`，轮廓识别度优先于细节数量。

应优先表达：

```text
主体外形
头部或核心区域
朝向
姿态
最具有辨识度的特征
```

不应优先表达：

```text
复杂纹理
过多阴影
微小装饰
无法辨识的零碎像素
```

---

## 3.4 像素轮廓语言

像素画边缘不是平滑曲线，而是经过设计的阶梯结构。

轮廓线应尽量使用有规律的步进，例如：

```text
1-1-1
2-2-1
3-2-1
```

避免随机抖动式轮廓：

```text
1-3-1-2-1-4
```

除非主体本身需要表现毛发、火焰、树叶或破损边缘。

轮廓设计应满足：

```text
边缘台阶有明确节奏
避免无意义凸起
避免单像素噪点
避免锯齿频率突然变化
尽量减少难以解释的凹陷
```

对于圆形、头部、眼睛、心形、按钮等规则图案，优先使用平衡且可预测的阶梯变化。

---

## 3.5 剪影优先

任何作品都应先通过纯剪影测试。

实现一个可选检查模式：

```bash
python scripts/render_pixel_art.py \
  examples/binary-cat-16x16.json \
  --output output/binary-cat.png \
  --preview-scale 8 \
  --silhouette-preview
```

输出：

```text
output/binary-cat_silhouette_preview_8x.png
```

剪影测试的目标是：

```text
去掉内部细节后，主体仍然大致可识别。
```

对于角色、动物和物体图标，优先保留最有辨识度的轮廓，例如：

```text
猫：耳朵、尾巴、坐姿
兔子：长耳朵
剑：剑柄和细长剑身
蘑菇：伞盖和短柄
史莱姆：圆润底部和顶部弧线
机器人：方形头部和机械结构
```

---

## 3.6 负空间设计

不要只关注填充区域，也要有意识地设计留白和镂空。

负空间用于表达：

```text
眼睛
嘴巴
四肢之间的间隙
武器与身体的分离
头发与肩膀之间的边界
图标内部的符号
```

二值图尤其依赖负空间。

例如，在黑白猫图标中，眼睛可以通过背景色镂空，而不是增加第三种颜色。

应避免：

```text
主体填充过密，导致内部结构消失
孔洞过多，导致轮廓破碎
镂空区域过小，放大预览中仍然难以辨认
```

---

## 3.7 单像素噪点控制

默认避免无意义的孤立像素。

孤立像素定义为：

```text
一个像素与上下左右四个方向均没有同色相邻像素。
```

可以实现一个辅助检查：

```bash
python scripts/validate_pixel_art.py \
  examples/binary-cat-16x16.json \
  --warn-isolated-pixels
```

孤立像素默认作为警告，不一定作为错误。

允许保留孤立像素的情况：

```text
眼睛高光
星星
火花
极小装饰
刻意制造的复古噪点
```

不允许保留的情况：

```text
无意义边缘凸点
误写入的像素
破坏轮廓节奏的杂点
无法解释的背景噪声
```

---

## 3.8 对称性规则

对于正面角色、图标、徽章、心形和简单物体，可以使用垂直对称或近似垂直对称。

支持：

```json
{
  "style": {
    "symmetry": "vertical"
  }
}
```

```json
{
  "style": {
    "symmetry": "approximate_vertical"
  }
}
```

```json
{
  "style": {
    "symmetry": "none"
  }
}
```

规则：

```text
vertical：
左右像素必须严格镜像。

approximate_vertical：
主体轮廓大体对称，但允许眼睛、阴影、耳朵、装饰和姿态存在少量差异。

none：
不做对称性要求。
```

不要为了追求对称而损害主体姿态。例如，侧身角色、奔跑动作和斜向武器不应强制镜像。

---

## 3.9 轮廓线规则

支持以下轮廓风格：

```text
none
solid
selective
```

### `none`

不额外绘制外轮廓。

适合：

```text
二值图标
极简符号
剪影画
```

### `solid`

主体外部使用统一轮廓色。

适合：

```text
四色 Sprite
复古游戏角色
小型怪物
道具图标
```

### `selective`

仅在需要分离主体与背景的位置绘制轮廓。

适合：

```text
颜色数量有限但需要更轻盈的角色 Sprite
局部高光
复杂姿势
```

在二值模式下，如果前景色本身已经构成剪影，则通常不需要额外轮廓。

---

## 3.10 阴影规则

支持以下阴影模式：

```text
none
flat
clustered
```

### `none`

不使用阴影。

适合：

```text
二值图标
8x8 图案
16x16 极简图标
```

### `flat`

使用单一阴影色块，不做渐变。

适合：

```text
四色像素画
Game Boy 风格
简化角色 Sprite
```

### `clustered`

使用成组的像素块表达阴影，不使用连续渐变。

适合：

```text
8 色角色头像
较大的 32x32 或 48x48 Sprite
```

任何阴影必须来自冻结调色板。

禁止：

```text
临时新增灰色
半透明阴影
抗锯齿阴影
模糊阴影
连续渐变
```

---

## 3.11 抖动规则

默认禁用抖动：

```json
{
  "style": {
    "dithering": "disabled"
  }
}
```

二值模式默认禁止抖动。

只有用户明确要求复古质感、阴影纹理或材质效果时，才允许使用规则化抖动：

```text
checkerboard
sparse_checkerboard
ordered_2x2
```

例如：

```json
{
  "style": {
    "dithering": "checkerboard"
  }
}
```

抖动像素必须：

```text
来自冻结调色板
具有规律
用于表达阴影或材质
不得变成随机噪点
不得破坏主体轮廓
```

禁止：

```text
随机散点抖动
误差扩散抖动
自动量化产生的抖动
用抖动掩盖轮廓设计问题
```

---

## 3.12 细节密度

支持：

```text
minimal
low
medium
high
```

推荐规则：

| 画布尺寸  | 推荐细节密度        |
| ----- | ------------- |
| 8x8   | minimal       |
| 16x16 | minimal 或 low |
| 24x24 | low 或 medium  |
| 32x32 | medium        |
| 48x48 | medium 或 high |
| 64x64 | high          |

不要在小画布中强行塞入过多细节。

当主体无法辨认时，优先删除装饰像素，而不是增加颜色或改变尺寸。

---

## 3.13 视觉重心

主体不应机械地填满画布。

需要根据用途预留边距：

```text
图标：
主体通常占画布宽高的 60% 至 85%。

角色 Sprite：
根据角色姿态决定，通常保留至少 1 至 2 像素安全边距。

头像：
头部可以占据较大面积，但不要紧贴全部边缘。

UI 符号：
轮廓应清晰，避免过度复杂。
```

推荐在配置中记录安全边距：

```json
{
  "composition": {
    "safe_margin": {
      "top": 1,
      "right": 1,
      "bottom": 1,
      "left": 1
    }
  }
}
```

安全边距属于风格建议，不是硬性限制。只有用户明确要求时，才作为校验错误处理。

## 3.15 主题类型与绘制策略

LLM 在绘制前，应根据主题选择对应策略。

### 图标

```text
优先剪影
减少内部细节
使用稳定负空间
尽量控制在中心区域
```

### 动物

```text
突出耳朵、尾巴、姿势
先决定侧面、正面或斜侧面
不要堆积毛发噪点
```

### 人物角色

```text
优先发型、脸部朝向、服装轮廓
小尺寸角色不追求五官写实
动作姿态比脸部细节更重要
```

### 怪物

```text
强化轮廓差异
允许不对称
使用少量夸张结构
保持重心明确
```

### UI 符号

```text
保持几何稳定
避免装饰性噪点
优先可读性
确保缩小后仍可识别
```

### OLED 和嵌入式点阵图

```text
严格二值
减少细线
减少孤立像素
避免过于复杂的孔洞
优先稳定显示
```

## 3.16 LLM 输出前的自检清单

在输出完整矩阵之前，LLM 应先在内部检查：

```text
1. 主体在指定尺寸中是否可识别？
2. 是否使用了未声明颜色？
3. 是否试图通过灰色或半透明像素柔化边缘？
4. 是否存在不必要的孤立像素？
5. 是否存在无意义的小凹陷或凸点？
6. 轮廓步进是否有规律？
7. 负空间是否有助于表达主题？
8. 小画布中是否加入了过多细节？
9. 风格预设是否与用途一致？
10. 是否保持原始画布尺寸不变？
```

---

## 3.17 LLM 生成矩阵时的建议流程

每次绘制按照以下阶段执行。

### 阶段 A：定义规格

输出：

```json
{
  "subject": "一只坐着的猫",
  "canvas": {
    "width": 16,
    "height": 16
  },
  "palette_limit": 2,
  "allow_transparency": false,
  "style": {
    "preset": "binary_icon"
  }
}
```

### 阶段 B：冻结调色板

输出：

```json
{
  "palette": [
    { "index": 0, "hex": "#FFFFFF", "name": "background" },
    { "index": 1, "hex": "#000000", "name": "foreground" }
  ]
}
```

### 阶段 C：设计剪影

先输出主体外形，不添加细节。

### 阶段 D：增加必要内部结构

增加眼睛、四肢间隙、衣服边界或负空间。

### 阶段 E：删除冗余像素

移除无意义噪点和破坏轮廓节奏的细节。

### 阶段 F：输出最终矩阵

执行完整校验并渲染 PNG。

## 3.18 局部修改时的风格约束

当用户提出修改时，优先使用 Patch，不要整张重画。

例如：

```text
把猫耳朵变尖一点。
让史莱姆更圆。
删掉右边多余的噪点。
眼睛向下移动一个像素。
轮廓太乱，整理一下边缘台阶。
```

Patch 修改必须遵循：

```text
不得改变画布尺寸
不得引入新颜色
不得隐式改变透明度
不得破坏冻结调色板
尽量只修改必要坐标
修改后重新校验
修改后重新渲染
修改后重新检查 PNG
```

## 3.19 风格化检查工具

扩展：

```text
scripts/validate_pixel_art.py
```

增加可选警告参数：

```bash
python scripts/validate_pixel_art.py \
  examples/binary-cat-16x16.json \
  --style-check
```

输出示例：

```text
VALID
Canvas: 16x16
Palette entries: 2
Palette limit: 2
Transparency: disabled

STYLE WARNINGS
- Isolated pixel detected at x=3, y=5.
- Irregular edge rhythm near x=11, y=7.
- Subject touches canvas boundary at x=0, y=8.
```

风格检查原则：

```text
颜色和尺寸错误：必须失败。
风格问题：默认警告。
```

不要将主观风格偏好全部做成硬性错误。

支持的风格警告至少包括：

```text
孤立像素
主体触碰画布边缘
过多单像素孔洞
轮廓局部突起
极端不平衡的视觉重心
对称模式下的明显镜像差异
```

```text

主体触碰边缘时可以产生警告。
孤立像素可以产生警告。
严格垂直对称模式下，明显不对称可以产生警告。
最近邻预览不得产生新颜色。
启用抖动时，仍然不得使用调色板外颜色。
二值模式默认禁用抖动。
terminal_oled 预设必须限制为严格二值模式。
```

---

## 3.22 最终原则

系统输出的作品应当满足：

```text
像素画不是低清晰度图片。
像素画不是缩小后的普通图片。
像素画不是自动量化后的图片。
像素画不是带有像素滤镜的生图结果。

每一个像素都必须是明确设计决策。
每一种颜色都必须来自冻结调色板。
原始尺寸必须始终与用户声明一致。
风格表达必须通过像素排列实现。
```
